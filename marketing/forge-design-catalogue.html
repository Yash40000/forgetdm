The problem

Continuous delivery needs continuous data

Hand-crafting test data every sprint is slow and unrepeatable. Copying production is fast but drags real PII into every lower environment. Design-driven data is the only approach that is both instant and safe.

Designed, not scavenged

Describe the exact shapes, edge cases and volumes you need as a reusable design — instead of hunting for a production row that happens to fit.

Repeatable on demand

Every design regenerates identical, in-domain data whenever a pipeline calls — no stale extracts, no queuing behind a central data team.

Safe by construction

Synthetic by design and masked by policy, so no test ever depends on real customer data — privacy compliance is built in, not bolted on.

Real where it must be

Accurate reference data, synthetic identities

Some values must be real to exercise business logic — clinical codes, tax brackets, routing numbers. ForgeTDM keeps those accurate from a governed reference set while replacing every personal identifier with safe synthetic data, so workflows behave exactly as in production without the risk.

  • Personal and private fields are masked or synthesised.
  • Referential integrity is preserved across every related table.

Questions

Frequently asked

Is this synthetic data or masked production data?+

Both, by design. You compose synthetic generators for identities and volumes, apply masking policies where you must start from real structure, and pull accurate reference values from a governed set — all in one design.

Does it keep foreign keys and relationships intact?+

Yes. Entities model the relationships once, and every recipe that uses them generates or subsets data that stays referentially consistent across all related tables.

Who can design versus who can request?+

Data architects model and publish trusted programs to the catalogue; testers browse and request them self-service. Maker-checker approval and reservations keep governance in control.

How does it fit our CI/CD pipeline?+

Any published design can be provisioned on demand or on a schedule, into a database, a download, an API target, or a virtual database — so the right data is waiting when a pipeline stage runs.

What about mainframe and legacy sources?+

Copybook-defined and relational sources are first-class. The same catalogue tiers apply, so a mainframe file design is composed and governed exactly like any other.
