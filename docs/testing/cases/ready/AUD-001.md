## Objective

Prove that every material create, update, delete, run, cancel, approval, promotion, and export is traceable, tamper-evident, searchable, and free of secrets or clear PII.

## Preconditions

- ADMIN, maker, checker, operator, and auditor users.
- Representative artifacts in each executable module and access to audit search, facets, stats, verification, and CSV export.

## Cases

| Case | Type | Action | Expected result and evidence |
|---|---|---|---|
| AUD-001-01 | CRUD | Create, update, and delete a data source, policy, blueprint, mapping, saved job, and security object. | One intelligible event per material state change includes actor, action, resource identity, outcome, timestamp, and correlation context. |
| AUD-001-02 | Execution | Start, complete, fail, retry, and cancel synthetic, provision, mapping, discovery, unstructured, and mainframe work. | Lifecycle events identify job/run and terminal outcome without logging generated or source records. |
| AUD-001-03 | Governance | Request, approve, reject, release, and promote governed artifacts. | Maker/checker identities, decision, comments, version, and outcome are attributable. |
| AUD-001-04 | Export | Export data, evidence, reports, and job packages. | Export event records actor, object, purpose/format/count where safe, and outcome before delivery. |
| AUD-001-05 | Denial | Trigger invalid login, access denial, validation failure, and dependency failure. | Security/material failures are auditable and sanitized; routine read noise does not bury them. |
| AUD-001-06 | Search | Query `/api/audit` by actor/action/outcome/resource/date and use facets/stats. | Filters, pagination, totals, and ordering are correct and stable. |
| AUD-001-07 | Export limits | Export audit CSV above page size and beyond the 5,000-event boundary. | Documented limit is enforced without truncation being mistaken for a complete export. |
| AUD-001-08 | Integrity | Run `/api/audit/verify`, then tamper with a disposable audit row in an isolated test database. | Clean chain verifies; tampering is detected at the affected sequence. |
| AUD-001-09 | Leakage | Scan audit database, API, CSV, and logs for passwords, JDBC secrets, tokens, cookies, clear PII, and payload rows. | No prohibited value is present. |
| AUD-001-10 | Authorization | Access audit UI/API as AUDITOR and a role without `audit.read`. | AUDITOR can read/export according to policy; unauthorized role receives `403` and cannot infer content. |

## Automation and Exit

- Maintain a coverage matrix mapping every material controller action to its expected audit event.
- Pass requires complete coverage, valid hash chain, and zero secret/clear-PII leakage.
