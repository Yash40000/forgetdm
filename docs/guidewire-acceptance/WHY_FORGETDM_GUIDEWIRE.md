# Why ForgeTDM for the Guidewire-Style Acceptance Scenario
ForgeTDM is valuable here because the hard problem is not producing random rows. It is coordinating realistic, private, referentially intact test data across Policy, Claims, Billing, and Contact applications while giving testers an understandable, repeatable operating flow.

## What This Scenario Demonstrates

- **One governed workflow:** connect, discover, generate, mask, subset, provision, validate, save, rerun, and audit from one UI.
- **Cross-database delivery:** the same operating model spans Oracle, SQL Server, and PostgreSQL without turning every test into a custom script project.
- **Relationship-aware data:** drivers, parent pulls, child cascades, per-edge traversal, and FK-safe generation protect realistic insurance graphs.
- **Deterministic privacy:** linked customer identities remain consistent while names, identifiers, accounts, addresses, and payment cards are protected.
- **Reusable self-service:** saved jobs let a tester rerun an approved design without rebuilding mappings or asking a database administrator to handcraft data.
- **Operational evidence:** table progress, partition progress, rejects, retries, approvals, and lineage make failures diagnosable and releases reviewable.
- **No inflated scale claim:** the workstation proves function and behavior; larger gates certify throughput on infrastructure sized for the target volume.

## Differentiation to Validate, Not Merely Claim

1. Faster time from a new schema to a reusable test-data job.
2. Clearer failure evidence at table, column, and partition level.
3. Stronger consistency across applications and databases.
4. Less dependence on hand-authored scripts for ordinary tester workflows.
5. A calmer, more explainable UI for complex data operations.
6. A single audit story from request through generated or provisioned result.

## Honest Boundaries

This acceptance run proves the tested engines, schemas, functions, volumes, and recovery paths. It does not by itself certify every Guidewire implementation, every vendor version, production HA/DR, or 5 TB throughput. Those become explicit certification-matrix entries backed by repeatable evidence.
