Table Profiles

Keep simple inclusion/filter settings here. Use Table Map for source, target, policy, and column mapping.

inclusion · filter · strategy
Column reference
  • Included — unchecked = completely excluded from the extraction run (no SELECT, no INSERT).
  • Filter (WHERE) — additional SQL WHERE clause applied after FK closure. Rows must satisfy both the FK path and this filter.
  • Strategy — Inherit: use global Q1/Q2. Follow Parent: rows constrained by FK join to parent PK set. Independent: an additional start table — seeded from its own Filter, then FK closure runs outward from it. Mark one or more tables Independent for a multi-start extraction; with at least one, a driver table is optional.

Referential Integrity

per-relationship traversal control

For each FK relationship (DB-catalog or custom) between included tables, set the traversal direction. Relationship-level rules take precedence over the global Q1/Q2 defaults.

Traversal direction reference
  • Both (Q1 + Q2) — traverse in both directions. Default for most relationships.
  • Q1 only — parent pull — pull parent rows the child references via FK. Do NOT expand child rows from parent. Use to limit explosion from large child tables.
  • Q2 only — child pull — pull child rows the parent owns. Do NOT traverse back up to parents from a child's perspective.
  • None — skip this relationship — decouple the two tables within this DataScope. FK exists in the schema but is irrelevant to this extraction.
Click "Refresh relationships" to load all FK edges for included tables.

Column Masking Overrides

literal values, suppress, null-out
Override types explained
  • Map / mask if rule exists - apply a saved policy rule when a policy is selected; otherwise copy the mapped value as-is.
  • Literal value — write a hardcoded constant (e.g. REDACTED, 0, 1900-01-01) instead of running a masking function.
  • Null out — always write NULL regardless of the original value or any policy.
  • Suppress column — exclude the column from both SELECT and INSERT. Target table must allow NULL or have a DEFAULT for this column.

VDB operations

Refresh restores the selected VDB from a dSource snapshot. Rewind restores the VDB to one of its own snapshots or bookmarks. ZFS rewind branches a new timeflow — nothing is destroyed.

LogSync — continuous WAL streaming

ZFS + Postgres

Enables point-in-time recovery between snapshots. ForgeTDM runs a pg_receivewal container with a dedicated replication slot that ships WAL continuously to the ZFS dataset. Enable on a ZFS dSource; provisioning with a Point-in-time timestamp will replay WAL to the exact second.

Masking scripts (Lua)

Optim-style user exits for anything the built-in functions don't cover. Reference a script from a rule as function SCRIPT with param1 = the script name. Scripts run sandboxed (no os / io / files) and must return the masked value. The forge.* helpers are deterministic (HMAC on the project secret), so custom masks keep referential integrity.
